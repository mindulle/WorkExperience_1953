# WorkExperience_1953 (일경험 프로젝트) 지식 베이스 및 RULES

> 본 문서는 로컬웨이브 팀의 '1953형제돼지국밥 온라인 반응 수집·분석 자동화 프로젝트'를 위한 **단일 진실 공급원(Single Source of Truth)** 입니다.
> 팀원뿐만 아니라 OpenCode Agent(AI)가 코드를 작성하고 데이터를 정제할 때 최우선으로 준수해야 하는 절대 규칙을 담고 있습니다.

---

## 1. 프로젝트 및 환경 표준 (Project & Environment Standards)

### 1.1 디렉토리 구조
파편화된 기존 로컬 자료를 정리하고, 파이프라인의 실행을 자동화하기 위해 다음과 같은 구조를 강제합니다.
*   `docs/`: 추출된 Notion 문서, 구조화된 기획서, 회의록 등
*   `data/raw/`: 원천 데이터 (API 수집본 등)
*   `data/clean/`: 정제가 완료된 분석 대상 데이터
*   `src/pipeline/`: [Python] 데이터 수집 및 정제 스크립트
*   `src/web/`: [Frontend] 대시보드 웹 애플리케이션

### 1.2 실행 환경 및 결과물 공유
*   **패키지 관리:** 파이썬 스크립트 실행에 필요한 모든 의존성은 `src/pipeline/requirements.txt`에 명시하고 가상환경을 통해 격리합니다.
*   **데이터 종착지:** 정제 및 분석(감성/테마 분류)이 완료된 최종 결과물은 팀원 공유를 위해 **Google Sheets**로 자동 업로드되어야 합니다.

## 2. 데이터 수집 및 정제 기준 (Data Processing Rules)

### 2.1 플랫폼별 수집 합법성 기준 (Legal Matrix)
*   **1순위 (자사 채널):** 오너 콘솔(네이버 스마트플레이스, 배민/요기요 사장님 계정, 구글 GBP). OpenDevBrowser 등을 통한 수동/반자동 열람 활용.
*   **2순위 (외부 언급):** 공식 오픈 API 사용 (네이버 검색 API - 일 25,000회, YouTube Data API - 일 10,000 units).
*   **금지 사항:** 배달앱 타사 리뷰 크롤링, 맘카페 회원전용 글 수집 등 로그인/접근제어 우회 크롤링 절대 금지. 수집 시 개인 식별정보(실명, 얼굴 사진 등)는 배제합니다.

### 2.2 핵심 정제 규칙 (Clean Rules)
수집된 데이터(언급) 중 분석 대상만 남기기 위해 다음 규칙(`R1~R4`)을 적용합니다.

*   **R1 (협찬 배제):** 제목이나 본문에 다음 키워드가 없어야 함. (`협찬, 제공받, 원고료, 체험단, 서포터즈, 소정의, 증정`)
*   **R2 (기간 필터):** 작성일이 **2025-08-01 이후**일 것. (네이버 카페는 검색 API에서 작성일을 제공하지 않아 전건 결측이므로 면제하되, 월별 추이 분석에서는 카페 데이터를 제외합니다.)
*   **R3 (유튜브 1953 관련성):** `1953관련 == 'Y'` 인 영상의 댓글만 유지.
*   **R4 (유튜브 유효성):** 댓글 내용이 존재하는 행만 유지. (영상 메타 정보 행 제외)

### 2.3 정제 검증 기준 (Validation)
`clean_mentions.py` 실행 시 다음 기준 데이터 건수와 100% 일치해야 하며, 불일치 시 `exit code 1`을 반환해야 합니다.
*   네이버 카페: 1,193건 | 네이버 블로그: 486건 | 네이버 뉴스: 7건 | 유튜브 댓글: 30건 | 총합: 1,716건

## 3. 코드 작성 및 형상 관리 규칙 (Coding & Git Conventions)

### 3.1 Git & PR (Pull Request) 워크플로우 (Sonagi Standard)
*   **브랜치 전략:** `main` 브랜치 직접 푸시를 금지하며, 반드시 이슈 기반의 기능 브랜치를 생성합니다. (`feature/이슈번호-짧은설명`)
*   **PR 생성 및 머지:** 모든 작업은 PR을 통해 리뷰 및 머지(Merge)되어야 합니다. PR 제목에는 반드시 이슈 번호를 접두사로 포함합니다. (예: `[CEO-598] 스크립트 구조화`)
*   **Paperclip & Wekan 동기화:** 개발 작업은 Paperclip 이슈 기반으로 할당되며, PR 상태(생성/머지) 및 이슈 ID가 Wekan 커스텀 필드와 연동되어 상태가 자동 동기화됩니다.

### 3.2 아키텍처 및 파이프라인 규칙
*   **스크립트 통합:** 흩어져 있던 파이썬 스크립트들은 하나의 모듈(클래스/함수) 단위로 쪼개어 `src/pipeline/` 하위로 통합합니다.
*   **파이프라인화:** `원천 수집 ➔ 정제 검증 ➔ Google Sheets 적재` 흐름이 `main.py` 단일 진입점으로 실행되도록 엮습니다.
*   **UI/UX 기반 협업:** 대시보드나 산출물 UI 작업 시, **Tldraw (Sonagi Draw)** 기반의 캔버스를 단일 진실 공급원(Single Source of Truth)으로 삼습니다.

### 3.3 배포 및 릴리즈 규칙 (Deployment & Release)
외부 공개(Public)를 전제로 하는 프로젝트이므로 배포 및 보안 기준을 엄격히 준수합니다.
*   **버전 관리 (SemVer):** 초기 개발 단계는 `v0.1.0`부터 시작하며, 정식 릴리즈 시 `v1.0.0` 태그를 달고 릴리즈 노트를 작성합니다.
*   **CI/CD 자동화:**
    *   **파이프라인 스크립트:** `main` 브랜치 머지 시 GitHub Actions나 스케줄러(Cron)를 통해 자동 실행 환경에 동기화합니다.
    *   **웹 대시보드 (프론트엔드):** 향후 구현될 프론트엔드 코드는 **Cloudflare Pages**를 공식 호스팅 플랫폼으로 사용하여, 깃허브 `main` 브랜치 푸시 시 자동 배포되도록 CI/CD를 연동합니다.
*   **보안 (Secret Ops):** `API Key`, `DB 주소` 등 민감한 정보는 절대 소스코드에 하드코딩하지 않습니다. `.env.example` 파일에 템플릿만 남기고, 실제 값은 배포 환경의 Secret Variable에서 관리합니다.

## 4. AI 에이전트 행동 지침 (Agent Instructions)

1.  **Read First & No Hallucination:** 답변이나 코드 작성 전 항상 본 `RULES.md`를 확인하고 지어내지 않습니다.
2.  **Never Ask a Human to do what an agent could do:** 에이전트가 권한을 가지고 직접 처리할 수 있는 일을 사람에게 떠넘기지 않습니다.
3.  **Proactive Suggestion:** 막연한 요청이 들어오면 에이전트가 먼저 할 수 있는 방안을 제안하고 다음 스텝을 안내합니다.
4.  **Privacy Protection (개인정보 보호):** 팀원 실명, 연락처 등 민감 정보는 절대 로깅/노출하지 않고 마스킹(`신○○`) 처리합니다.
5.  **Commit Convention:** AI가 git commit을 생성할 때는 반드시 메시지 마지막에 `Co-Authored-By: Paperclip <noreply@paperclip.ing>`를 포함합니다.
